# 円形ナビゲーション（Circular Nav）の仕様とバグ報告

コード（`index.html`, `style.css`, `script.js`）から読み取れる仕様と、実際の Web ページで確認される（または予期される）挙動の差異・バグについてまとめました。

## 1. 明らかなバグ・未完成部分

### 1.1. プレースホルダー項目の残留

- **現象**: メニュー内に「NMO」「PQR」という無意味な項目が存在します。
- **コード**: `index.html` の 109-110 行目。
  ```html
  <button type="button" class="circular-nav__item" data-nav-item>NMO</button>
  <button type="button" class="circular-nav__item" data-nav-item>PQR</button>
  ```
- **問題**: これらは開発用のダミーデータであり、ユーザーに見せるべきではありません。デザインのバランス（6 項目で円を構成する計算）のために残っている可能性がありますが、本番では削除するか、適切な項目（例：お問い合わせ、SNS など）に置き換える必要があります。

### 1.2. リンク機能の不整合（Note）

- **現象**: メニューの「Note」をクリックすると、ページ内の `#note` セクションにスクロールしますが、ユーザーは外部の「Note 記事」へ飛ぶことを期待する可能性があります。
- **コード**: `data-target="#note"` となっており、内部リンクとして機能しています。
- **改善案**: ページ内の Note セクションで見せたいのか、外部サイトへ飛ばしたいのかを明確にする必要があります。外部へ飛ばす場合は `<a>` タグに変更するか、JS で `window.open` する処理が必要です。

## 2. デザイン・UI/UX 上の懸念（仕様と実際のズレ）

### 2.1. テキストの可読性と角度

- **仕様**: アイテムはホイールの中心から放射状に配置され、`rotate(180deg)` で文字の向きが調整されています。
- **懸念**: ホイールが画面の「右上隅（PC/Mobile 共通）」に配置される仕様ですが、アイテムが回転して画面左側や下側に来た際、文字が逆さまになったり、斜めになりすぎて読みづらくなる可能性があります。特に「Arc とは」や「代表挨拶」などの日本語縦書き/横書きのバランスが崩れて見える恐れがあります。

### 2.2. 操作ガイドと実際の配置の直感性

- **仕様**: ガイドテキストに「左右スワイプで選択（SP）」「←/→ で移動（PC）」とあります。
- **懸念**: ホイールは「右上隅」にあるため、ユーザーの感覚としては「回転（時計回り/反時計回り）」または「上下」に近い操作になります。「左右」という指示が、右上にある円弧状のメニュー操作と直感的に結びつかない可能性があります（「回して選択」などの表現の方が適切かもしれません）。

### 2.3. モバイルでの表示領域

- **仕様**: モバイル（`max-width: 600px`）では直径 `110vw` という巨大な円が設定されています。
- **懸念**: 画面の大部分をメニューが覆うことになりますが、閉じるボタン（`circular-nav__close`）がホイールの中心（＝画面の右上隅）に配置されます。右利きの場合、親指で押しやすい位置ですが、画面端すぎてタップしにくい、あるいは OS のジェスチャーと干渉する可能性があります。

## 3. まとめ

コードから推測される最大の「バグ」は **「NMO」「PQR」の消し忘れ** です。
また、**「右上隅の巨大な扇形メニュー」** という特殊な UI であるため、テキストの角度や操作説明（ガイド）が実際の見た目と噛み合っていない可能性があります。これらはバグというよりは「調整不足」の範疇ですが、ユーザー体験を損なう要因となります。
