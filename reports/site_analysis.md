# Arc Yamanashi プロトタイプサイト分析レポート

## 1. サイト概要

本サイトは「Arc Yamanashi」という団体またはプロジェクトのプロトタイプサイトです。「Arc（円弧、架け橋）」をテーマに、地域とのつながりや物語（ストーリーアーク）を表現しています。
一般的な Web サイトの構成とは異なり、スクロール連動のアニメーションや独自の円形ナビゲーションを採用した、没入感のあるデザインが特徴です。

## 2. コンテンツ構成

サイトはシングルページ構成で、以下のセクションで成り立っています。

1.  **ヒーローセクション (Hero)**

    - **ビジュアル**: 歩く人物のシルエット、回転する円形のロゴ配置、中央のキャッチコピー。
    - **コピー**: 「放課後、地域で奮闘中...」というタグラインがアニメーション（「...」が動く）付きで表示。
    - **機能**: スクロール量に応じて画面が切り替わる独特な挙動。

2.  **About セクション (Arc とは)**

    - **コンセプト**: 「小さな一歩の積み重ねが、やがて大きな輪となるように」というメッセージ。
    - **由来**: 「Arc」＝「孤（個）」が集まって「縁（円）」になる、物語の「ストーリーアーク」という意味が込められている。
    - **メンバー紹介**: Haruto 氏、Riko 氏の写真と想い（「運が導いた出会い」「地域に笑顔と縁を」）。

3.  **Note セクション**

    - **外部連携**: Note 記事への誘導リンク。カード型デザインで「私たちは活動しています」とアピール。

4.  **ナビゲーション**
    - **Circular Nav**: 画面全体を使った円形のホイール型メニュー。

## 3. デザイン・レイアウト・UI/UX

### デザインの方向性

- **ミニマルかつ大胆**: 白（`#f8f9fb`）と黒・濃紺（`#111827`）を基調とした高コントラストな配色。
- **和文タイポグラフィ**: Noto Sans JP を使用し、余白を多めに取った読みやすいレイアウト。
- **円のモチーフ**: ロゴ、ナビゲーション、装飾に至るまで「円（Arc）」が徹底して使われています。

### 特徴的な UI/UX

1.  **スクロールジャック風のヒーロー体験**

    - `style.css` の `.hero-scroll` クラスに見られるように、通常のスクロールで画面がそのまま下に移動するのではなく、スクロール量（`--hero-progress`）に応じて要素が移動・フェードアウトする演出が入っています。これにより、ファーストビューで強い印象を与えています。

2.  **Circular Navigation (円形ナビゲーション)**

    - **独自性**: 一般的なリスト型メニューではなく、画面右下に中心を持つ巨大なホイールを回すようなインターフェース。
    - **操作感**: ドラッグ（スワイプ）で回転させ、項目を選択するアナログな操作感（`script.js` で物理挙動に近い慣性やスナップ処理を実装）。
    - **遊び心**: 実用性よりも「触って楽しい」体験を重視しています。

3.  **ダイナミックなロゴ**
    - 背景色に応じてロゴの色（黒/白）が自動で切り替わる処理（`evaluateLogoContrast`）が入っており、視認性を担保しつつデザインを崩さない工夫がされています。

## 4. アニメーション・演出（作者のこだわり）

作者が特にこだわったと思われるポイントです。

1.  **「落下する人物」シミュレーション (`fall-sim.js`)**

    - **隠し要素**: ヒーローセクションの円が回転し、特定の角度（欠けた部分が下に来た時など）になると、中央の「歩く人物」が物理演算で落下し、下のロゴに着地してバウンドするアニメーションが実装されています。
    - **技術**: 簡易的な物理エンジン（重力、反発係数）を JS で書いており、ただのアニメーション GIF などではなく、コードで動きを作っている点に強いこだわりを感じます。

2.  **スクロール連動フェード (`initializeScrollReveal`)**

    - 要素が画面内に入ってくるときに、単に表示されるのではなく、スクロール量に応じて滑らかに浮き上がるような演出。`IntersectionObserver` ではなく、スクロール位置を計算して `opacity` と `transform` を細かく制御しています。

3.  **円形メニューの物理挙動**
    - メニューを回した時の「慣性」や、項目が選択位置に吸い付く「スナップ」処理など、ネイティブアプリのような触り心地を目指しています。

## 5. 修正・改善が必要な点（未完成・バグ・UX 課題）

現状のコードと挙動から推測される課題点です。

### 未完成・プレースホルダー

- **メニュー項目**: ナビゲーションに「NMO」「PQR」という無意味なラベルがあり、リンク先も設定されていません。
- **画像**: `images/IMG_1.jpg` など、ファイル名が仮のままである可能性があります。
- **Note リンク**: リンク先が `https://note.com/yamanashi_hs` となっていますが、実在するか確認が必要です（プロトタイプ用の仮 URL の可能性）。

### UI/UX の課題

1.  **ナビゲーションの発見しやすさ (Discoverability)**
    - ハンバーガーメニューを開いた後、円形メニューの操作方法（ドラッグして回す）が直感的でない可能性があります。ガイド（「左右スワイプで選択...」）は出ますが、初見では戸惑うユーザーがいるかもしれません。
2.  **アクセシビリティ**
    - 独自の円形メニューはキーボード操作（矢印キー）に対応しているようですが、スクリーンリーダーでの順次読み上げやフォーカス順序が適切か検証が必要です。
3.  **スクロールの重さ**
    - `window` の `scroll` イベントで `requestAnimationFrame` を回してスタイルを更新しているため、低スペックなデバイスでは描画がカクつく可能性があります。

### バグ・実装上の懸念

- **`fall-sim.js` の挙動**: 人物が落下した後、スクロールして戻った場合やリサイズした場合に位置がずれたり、意図しない場所に留まったりする可能性があります（リセット処理はありますが、複雑な状態遷移を含んでいます）。
- **ブラウザ互換性**: `clip-path` や高度な CSS 変数の計算（`calc`の多用）を行っているため、古いブラウザ（特に Safari の旧バージョンなど）でレイアウト崩れが起きるリスクがあります。

## まとめ

非常に野心的で、技術力とデザインセンスを感じさせるプロトタイプです。特に「円」を軸にした UI とアニメーションへのこだわりは素晴らしく、完成すれば非常に印象的なサイトになるでしょう。
一方で、独自の UI はユーザビリティとのトレードオフになりがちです。今後は「操作のわかりやすさ」や「パフォーマンス」をチューニングしつつ、プレースホルダー部分を埋めていくフェーズにあると言えます。
