## 重要

このフォルダは vite 構成です。
すべての出力は**必ず**日本語で行うこと
このウェブサイトは**スマホファースト**。モバイルのレイアウトに注目して編集すること。
ユーザーが〜したいと言ったときはユーザーの意図を汲み取るためにそれは〜ということですか？と一度確認すること。また、その際コードの編集は禁止。ユーザーがはいと言った時、もしくは実装と言った時のみ、その仕様で実装すること。ユーザーから明確な指示がない限り、コードの編集は**禁止**。
コードの編集をした後は必ず chrome devtools を用いて、レイアウトを確認すること。ただし、3 回以上エラーが出た場合はそれを報告するために中断すること。

# === 禁止事項 ===

1. コード編集について

   - ユーザーが編集してくださいと明確に指示しない限り、コードの編集は行わないこと。ただし、コードの閲覧は自由。

2. **ビジュアルデザインの改変禁止**

   - このセッションの命令は全てユーザーからの明確な指示がない限りという制約を受ける。明確な指示がある場合は禁止事項は無効化される。
   - 色（背景・文字・線・影など）の追加・変更を行わない。
   - フォント、文字サイズ、文字間隔、余白、角丸、ボーダー形状、アニメーション、ブラー、グラデーションの変更を行わない。
   - 背景色を重ねたり、黒背景＋白背景のようなレイヤー構成を作らない。
   - 新しい図形・シェイプ・装飾を生成しない。

3. **テキスト・要素の創作禁止**

   - 指定されたテキスト以外の新しいテキスト（文言・見出し・ラベル・説明文）を追加しない。
   - プレースホルダ（例："Sample"・"Hello"など）を挿入しない。
   - 既存テキスト・画像・リンク先を改変しない。

4. **ファイル操作の逸脱禁止**

   - 指定されたファイル以外を編集しない。
   - 新しいファイル・フォルダを作成しない。
   - ファイル名・パスを変更しない。

5. **説明文・コメント出力の禁止**

   - 「ここを変更しました」等のコメントをコードファイル内に挿入しない（指定がある場合を除く）。
   - コードファイル内に説明文を追加しない。
   - ユーザーが CSS などに対して、要素をこれくらい大きくしてください。などと指示し、編集するときは calc()などを用いて、+数値などで表現せず、直接数値を変更すること。
   - CSS を編集した後にコメントで〜〜をしましたなどの説明文はどうしても必要な時を除いて絶対に行わないこと。

6. **レポート・ドキュメントの保存場所**

   - 生成した Markdown レポートやドキュメントは、すべて `reports` ディレクトリに保存すること。

7. **言語設定**
   - 生成するレポート、ドキュメント、コミットメッセージ等はすべて**日本語**で記述すること。
