# モバイル UI/UX：回転と配置のズレに関する詳細分析

ユーザー様よりご指摘のあった「回転時のテキストのずれ」と「アクティブ位置のズレ」について、**「なぜそうなっているのか（設計意図）」** と **「実際の見え方への影響」** を中心にまとめました。

## 1. テキストの回転ずれ（Rotation Misalignment）

### 現象

アクティブ（選択中）の項目が、水平（0 度）ではなく、**約 -23 度（右上がり）** に傾いています。

### 設計意図（推測されるこだわり）

この傾きは、バグではなく**「放射状（Radial）デザイン」** を意図した結果である可能性が高いです。

- **円の中心に向かう視線**: メニューは画面右上（ホイール中心）から放射状に広がっています。テキストをその「半径（Radius）」のラインに合わせることで、円の一部として自然に馴染ませる意図があります。
- **ダイナミズム**: 完全に水平にするよりも、少し角度をつけることで「動き」や「円環」を感じさせる演出になります。

### 実際の見え方と課題

- **-23 度という角度**: 放射状に配置する場合、理論上は「半径の角度（165 度）」に合わせて反転（+180 度）させた **-15 度** が「正対」する角度です。
- **ズレの正体**: ここに `displayOffset: -8度` が加わることで、正対位置からさらにズレて **-23 度** になっています。
- **結果**: 「円の中心に向いている（Radial）」ようにも、「水平（Horizontal）」ようにも見えない、**「どっちつかずの角度」** に見えてしまっている可能性があります。これが「ズレ」として認識される原因です。

## 2. アクティブ位置のズレ（Position Offset）

### 現象

「見た目でちょうど良い位置」と「文字サイズが最大化される位置」が食い違っています。

### 設計意図（推測されるこだわり）

- **指の被り対策**: スマホ操作時、指で隠れる部分を避けて、少しずらした位置にテキストを表示させようとした可能性があります（`displayOffset: -8`）。
- **視覚的重心の補正**: 日本語フォントや文字数によって、数学的な中心と見た目の中心が異なるため、それを微調整するためのオフセット値である可能性があります。

### 実際の見え方と課題

- **操作の乖離**: ユーザーは「文字が見やすい位置（表示位置）」に指を止めようとしますが、システムは「そこから 8 度ずれた位置（論理位置）」で判定を行っています。
- **結果**: 「真ん中に合わせたのに最大にならない」「通り過ぎてから最大になる」という、**操作に対するフィードバックの遅れ（ラグ）** のように感じられます。これが「気持ちよさ」を損なっている要因です。

## 3. 結論と改善の方向性

現状の挙動は、単なるバグではなく**「円形デザインへのこだわり（放射状配置）」** と **「操作性への配慮（オフセット）」** が、計算上で噛み合わずに**「視覚的な違和感」** を生んでしまっている状態です。

### 改善案

「スマホ前提」かつ「美しさ」を重視する場合、以下の調整を推奨します。

1.  **角度の「意図」を明確にする**

    - **水平にするなら**: アクティブ時は強制的に `rotate(0deg)` に近づける補正を入れる。
    - **放射状にするなら**: `displayOffset` 分のズレを回転角度から引いて（あるいは足して）、**「完全に円の中心を向く角度（-15 度付近）」** に修正する。

2.  **判定と表示の一致**
    - `displayOffset` を判定ロジックにも組み込み、「見た目の中心」に来た瞬間に「アクティブ判定」が入るように計算式を統一する。これにより、操作のダイレクト感（吸い付き感）が向上します。
